Which Helper Should I Use?

This page is a routing guide. Pick the narrowest documented helper that already fits the workflow. If none fits cleanly and the use case is repeated, submit a proposal instead of mutating the shared helper contract ad hoc.

Need Use Why Demo
Read-only message uiAlert(...) Fast semantic dialog without rebuilding modal actions. UI Utilities
Confirm / cancel decision uiConfirm(...) Shared confirm semantics, variants, and icons. UI Utilities
Prompt for one input uiPrompt(...) Short value collection without building a full modal form. UI Utilities
Short custom modal form createFormModal(...) Strict row model, field errors, form error, and helper-owned busy submit flow. Form Modal
Shared login flow createLoginFormModal(...) Consistent structure with field-name remapping for backend compatibility. Form Modal
Session re-auth createReauthFormModal(...) Locked identifier and shared re-auth wording/behavior. Form Modal
Status change workflow createStatusUpdateFormModal(...) Shared structure with app-supplied status vocabulary. Form Modal
Categorized reason-required flow createReasonFormModal(...) Reason select plus details textarea for monitoring/audit use cases. Form Modal
Transient status feedback createToastStack(...) Shared semantic icons, optional speech, and auto-dismiss behavior. UI Utilities
Flat tabular data createGrid(...) Sorting, query state, pagination, virtualization, and dense action cells. Grid
Hierarchical data with columns createTreeGrid(...) Hierarchy plus columns, tree-aware search, and lazy loading. Tree Grid
Hierarchy shown visually createHierarchyMap(...) Hierarchy-first visual map with overlay relationships and pan/zoom. Hierarchy Map
Standalone media viewing createMediaViewer(...) Dedicated image/video viewer with fit and navigation behaviors. Media Viewer
