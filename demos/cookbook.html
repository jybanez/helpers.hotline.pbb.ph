Operational UI Cookbook

Use this page to find the right helper flow by common operational task rather than by raw component name. These recipes are intentionally narrow. If a workflow stops fitting the shared pattern cleanly, fall back to the lower-level helper instead of overloading the wrapper.

Common Modal Workflows

Login

Use createLoginFormModal(...) for shared login structure, busy submit handling, and field-name remapping. Open Form Modal Demo

Re-auth

Use createReauthFormModal(...) when a session-sensitive flow requires password confirmation without rebuilding the form. Open Form Modal Demo

Status Update

Use createStatusUpdateFormModal(...) for routine state changes with status options, remarks, and optional notify behavior. Open Form Modal Demo

Reason Required

Use createReasonFormModal(...) when you need a categorized reason select plus details for audit and reporting. Open Form Modal Demo

Alert / Confirm / Prompt

Use uiAlert(...), uiConfirm(...), or uiPrompt(...) for short decision flows instead of hand-built modals. Open UI Utilities Demo

Busy Async Form

Use createFormModal(...) or dialog wrappers with helper-owned busy state. Do not rebuild duplicate-submit locking in app code. Open UI Utilities Demo

Feedback And Messaging

Toast Feedback

Use createToastStack(...) for semantic transient feedback, shared status icons, and optional speech. Open UI Utilities Demo

Semantic Dialogs

Use dialog variants and descriptions when you need stronger status framing without building custom dialog chrome. Open UI Utilities Demo

Data Exploration

Flat Data Grid

Use createGrid(...) for flat tabular data, remote query flows, and dense action cells. Open Grid Demo

Tabular Hierarchy

Use createTreeGrid(...) when you need hierarchy with aligned columns, tree-aware search, and virtualization. Open Tree Grid Demo

Visual Hierarchy

Use createHierarchyMap(...) when the hierarchy needs to be read visually and overlay links matter. Open Hierarchy Map Demo
